# FleetVision 專案背景

## 專案目的

FleetVision 旨在建立一套共享車輛借還車情境下的車輛外觀車損辨識流程。使用者在借車與還車時拍攝車輛外觀照片，系統透過模型偵測車損位置，並在未來與借還車前後照片比對流程整合，協助判斷是否出現疑似新增車損。

## 第一版目標

第一版不追求完整自動理賠，而是完成以下能力：

1. 管理大量車況照片。
2. 篩選外觀照片。
3. 建立車損標註資料。
4. 訓練 YOLOv8 Detect 單類別車損模型。
5. 輸出 bbox、confidence、model_version。
6. 寫入 PostgreSQL。
7. 用 Dashboard 展示。
8. 建立可擴充的新增車損比對邏輯。

## 為什麼不直接做索賠模型？

目前 claimable damage 樣本數偏少，且索賠與否不只是視覺問題，還涉及業務規則、損傷位置、大小、既有紀錄、人工覆核與公司政策。因此第一版應先做可見車損偵測，索賠候選與人工覆核放在後續規則層。
